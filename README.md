# Ramblemaxxer

> Stop guessing, start maxxing. Schedule optimization for Loyola University Chicago students.

---

## Stack

| Layer | Tech | Why |
|---|---|---|
| Frontend | Vite + React | Fast dev, static build for CPanel |
| Backend | Node.js + Express | Familiar, runs via CPanel Passenger |
| Database | SQLite (better-sqlite3) | Zero config, single file, backs up trivially |
| Auth | express-session + bcrypt | Simple, no external services |

## Local Development

### Prerequisites
- Node.js 18+
- npm 9+

### First-time setup

```bash
# 1. Clone the repo
git clone <your-repo-url>
cd ramblemaxxer

# 2. Install root dependencies (just concurrently)
npm install

# 3. Install server dependencies
cd server && npm install && cd ..

# 4. Install client dependencies
cd client && npm install && cd ..

# 5. Initialize the database
npm run db:init

# 6. Seed Penelope's data
npm run db:seed

# 7. Start both servers
npm run dev
```

This starts:
- Express API on **http://localhost:3001**
- Vite dev server on **http://localhost:5173** (with `/api` proxied to Express)

Open **http://localhost:5173** in your browser.

**Penelope's test login:** `penelope@ramblemaxxer.local` / `changeme`

## Invite Flow (adding Penelope's friends)

1. Log in as Penelope
2. POST to `/api/auth/invite` (or use the UI when it's built)
3. Share the invite URL: `https://ramblemaxxer.com/register?token=<token>`
4. Friend visits the link, creates their account
5. Tokens expire in 7 days and are single-use

## CPanel Deployment (Namecheap)

### One-time setup

1. **SSH into your hosting account** (Namecheap → cPanel → Terminal, or SSH)

2. **Upload your files** (via Git or FTP):
   ```bash
   # Recommended: set up a private Git repo and pull from server
   cd ~/
   git clone <your-repo-url> ramblemaxxer
   cd ramblemaxxer
   ```

3. **Install dependencies on the server:**
   ```bash
   cd ~/ramblemaxxer
   cd server && npm install --production && cd ..
   cd client && npm install && cd ..
   ```

4. **Build the frontend:**
   ```bash
   npm run build
   # This puts built files in server/public/
   ```

5. **Initialize the database:**
   ```bash
   npm run db:init
   npm run db:seed
   ```

6. **Configure Node.js app in cPanel:**
   - cPanel → Software → Setup Node.js App
   - Node.js version: 18+
   - Application mode: Production
   - Application root: `/home/<username>/ramblemaxxer`
   - Application URL: `ramblemaxxer.com`
   - Application startup file: `app.js`
   - Set environment variable: `SESSION_SECRET=<random-string>`
   - Set environment variable: `APP_URL=https://ramblemaxxer.com`
   - Click **Create** / **Save**

7. **Point your domain:**
   - In Namecheap DNS, point `ramblemaxxer.com` to your hosting IP

### Deploying updates

```bash
# On the server:
cd ~/ramblemaxxer
git pull
cd client && npm run build && cd ..
# Restart the app in cPanel → Node.js Apps → Restart
```

### Important notes for CPanel
- The SQLite database file lives at `server/db/ramblemaxxer.db` — **back this up regularly**
- CPanel's Passenger keeps the Node process alive; if it crashes, restart from cPanel UI
- Logs: cPanel → Errors, or check `~/logs/`
- `SESSION_SECRET` must be set as an env var in cPanel — never commit it to git

## Environment Variables

| Variable | Default | Required in prod |
|---|---|---|
| `PORT` | 3001 | No (Passenger sets it) |
| `NODE_ENV` | development | Yes (`production`) |
| `SESSION_SECRET` | dev-secret-... | **Yes** |
| `APP_URL` | http://localhost:5173 | Yes (for invite links) |

## Data Files

The `data/` directory contains the parsed course catalog and degree requirements. These are read-only at runtime — the solver references them but students' personal data lives in SQLite.

To refresh the catalog from Loyola's PDFs, run the parser:
```bash
node scripts/parse-catalog.js
```
